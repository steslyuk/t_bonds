# Деплой Telegram-бота на Railway

Бот собирает портфели облигаций на данных T-Invest. Работает в режиме polling
(фоновый worker), публичный URL не нужен.

## Файлы проекта

```
telegram_bot.py      # сам бот (тонкая обёртка)
tinvest_bond_bot.py  # ядро: данные + оптимизатор + прогноз (НЕ менять)
requirements.txt     # зависимости
Procfile             # worker: python telegram_bot.py
railway.json         # настройки деплоя (start command, рестарт)
```

## Шаг 1. Токен бота
1. В Telegram напиши [@BotFather](https://t.me/BotFather) → `/newbot`.
2. Задай имя и username. BotFather выдаст **TELEGRAM_TOKEN** вида `12345:AA...`.

## Шаг 2. Токен T-Invest
Используй **readonly**-токен T-Invest (вывод денег и торговля им невозможны).
Это переменная **TINVEST_TOKEN**.

## Шаг 3. Залить код на GitHub
Создай репозиторий и положи туда все 5 файлов выше. Railway деплоит из GitHub.

## Шаг 4. Railway
1. [railway.com](https://railway.com) → **New Project** → **Deploy from GitHub repo** → выбери репозиторий.
2. Railway сам поставит зависимости из `requirements.txt` и запустит `python telegram_bot.py`.
3. Открой вкладку **Variables** и добавь:
   - `TELEGRAM_TOKEN` = токен от BotFather
   - `TINVEST_TOKEN` = readonly-токен T-Invest
   - `ALLOWED_USERS` = (пока пусто — заполнишь на шаге 5)
   - (необязательно) `MAX_CANDIDATES` = `60`, `INCLUDE_QUAL` = `0`
4. Если Railway создал **web**-сервис и жалуется на порт — это нормально: бот не слушает порт.
   В настройках сервиса убедись, что start command = `python telegram_bot.py`
   (он уже прописан в `railway.json` и `Procfile`). Домен/порт не нужны.

## Шаг 5. Ограничить доступ (важно!)
Бот работает на **твоём** токене T-Invest, поэтому открывать его всем нельзя.
1. После первого деплоя открой бота в Telegram → `/id` → он покажет твой Telegram id.
2. Впиши id в переменную `ALLOWED_USERS` (через запятую, если несколько).
3. Railway перезапустит бота. Теперь команды работают только у тебя.

> Пока `ALLOWED_USERS` пуст — бот отвечает всем (в логах будет предупреждение). Заполни обязательно.

## Шаг 6. Проверка
В Telegram:
```
/start                 — справка
/demo 1500000 3        — портфели на синтетике (работает даже без TINVEST_TOKEN)
/p 1500000 3           — реальные данные, сравнение 3 профилей + кнопки
/p 1500000 3 mod 18    — умеренный профиль, реинвест 18%
1500000 3 agg          — то же без команды
```

## Заметки
- **Кеш.** Реальная вселенная бумаг кешируется на 5 минут, чтобы не дёргать API на каждый
  запрос и не упираться в лимиты. Цены за это время «подмерзают» — для планировщика ок.
- **Скорость.** Первый реальный запрос на новый срок тянет купоны и стаканы по десяткам бумаг —
  до минуты. Дальше из кеша мгновенно. Регулируется `MAX_CANDIDATES`.
- **Рейтинг.** Сейчас ОФЗ → AAA автоматически, остальным — заглушка (`RATING_MAP` в
  `tinvest_bond_bot.py` + `--unknown-rating`). Пока не подключишь реальный рейтинг/скоринг,
  умеренный и консервативный профили на реальных данных будут неточны. Это место под твой PD-скор.
- **Стоимость.** Railway тарифицирует по потреблению; лёгкий polling-бот стоит копейки,
  но на бесплатном триале есть лимиты по часам — следи за балансом.
- Бот считает **план**, заявки не выставляет. Это не индивидуальная инвестиционная рекомендация.
